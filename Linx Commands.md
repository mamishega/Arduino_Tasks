# Linux Commands

| Command            | Full Syntax Example                     | Description                                                                                                                        | Purpose in This Task                                              |
| ------------------ | --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `mkdir`            | `mkdir IT_Maintenance`                  | Creates a new directory (folder) in the current location.                                                                          | Used to create a project folder to organise maintenance files.    |
| `ls`               | `ls -l`                                 | Lists files and directories in the current directory. The `-l` option shows detailed information (permissions, size, owner, date). | Used to verify folder and file creation.                          |
| `cd`               | `cd IT_Maintenance`                     | Changes the current working directory to another directory.                                                                        | Used to navigate into the created folder.                         |
| `nano`             | `nano maintenance_log.txt`              | Opens a text file in the Nano text editor. If the file does not exist, it creates one.                                             | Used to create and edit the maintenance log file and script file. |
| `cat`              | `cat maintenance_log.txt`               | Displays the contents of a file in the terminal.                                                                                   | Used to verify file contents after saving.                        |
| `sudo`             | `sudo apt update`                       | Executes a command with superuser (administrator) privileges.                                                                      | Required for system updates and software installation.            |
| `apt update`       | `sudo apt update`                       | Refreshes the package list from repositories.                                                                                      | Ensures the system has the latest package information.            |
| `apt full-upgrade` | `sudo apt full-upgrade -y`              | Upgrades all installed packages, including dependencies. `-y` auto-confirms prompts.                                               | Used to fully update the operating system.                        |
| `apt autoremove`   | `sudo apt autoremove -y`                | Removes unused packages and dependencies.                                                                                          | Cleans the system and frees storage space.                        |
| `apt install`      | `sudo apt install gimp -y`              | Installs software packages from repositories.                                                                                      | Used to install GIMP and LibreOffice.                             |
| `reboot`           | `sudo reboot`                           | Restarts the system immediately.                                                                                                   | Used after system updates.                                        |
| `date`             | `sudo date -s "2026-02-21 10:00:00"`    | Sets the system date and time manually.                                                                                            | Used to configure correct system time.                            |
| `chmod`            | `chmod +x auto_maintenance.sh`          | Changes file permissions. `+x` makes a file executable.                                                                            | Used to make the maintenance script runnable.                     |
| `./`               | `./auto_maintenance.sh`                 | Executes a script located in the current directory.                                                                                | Used to manually run the maintenance script.                      |
| `crontab`          | `crontab -e`                            | Opens the cron table editor to schedule tasks.                                                                                     | Used to schedule automated maintenance.                           |
| `crontab -l`       | `crontab -l`                            | Lists all scheduled cron jobs.                                                                                                     | Used to verify that the maintenance task is scheduled.            |
| `echo`             | `echo "Starting System Maintenance..."` | Prints text to the terminal or writes text to a file.                                                                              | Used inside the script to display status messages.                |
| `>>`               | `>> maintenance_log.txt`                | Appends output to the end of a file without overwriting it.                                                                        | Used to log maintenance completion date.                          |
